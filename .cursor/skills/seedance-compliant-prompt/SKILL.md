# Seedance 2.0 文生视频提示词（合规版）

将剧本、分镜或粗糙描述改写为符合 [Doubao Seedance 2.0 系列提示词指南](https://www.volcengine.com/docs/82379/2222480?lang=zh) 要求、且可通过筷子 OpenAPI 审核的**纯文本**提示词。

## 何时启用

- 用户要写/改文生视频提示词
- 批量从剧本生成每集视频提示词
- API 返回版权、合规、审核类错误
- 输入是剧本格式（△、角色表、对白行）
- 项目内 `TextToVideoModal`、生成视频步骤批量生成前的提示词处理

## 官方要求摘要（火山方舟 Seedance 2.0）

Seedance 2.0 面向**多模态**输入（图/视频/音频/文本）。方舟文档强调：提示词应像**导演分镜稿**，写清镜头能拍到的可见事实，并与所选**时长**匹配。

### 高质量提示词公式

按官方结构组织（纯文本场景下去掉 @ 引用，其余保留）：

```text
[主体/人物设定] + [场景/环境] + [动作/运动描述] +
[运镜语言] + [分时段描述（≥8秒推荐）] + [转场/特效] +
[音频/音效设计（可选）] + [风格/氛围]
```

### 四层要素（写作时逐层补全）

| 层级 | 写什么 | 要点 |
|------|--------|------|
| **主题与节拍** | 这一镜要展示什么 | 一句话核心；5 秒聚焦**一个**节拍或短蒙太奇 |
| **视觉风格** | 画面质感与色调 | 电影感/写实、光照、色调、景深、画幅意图 |
| **画面与动态** | 可见陈设、人物动作、镜头运动 | 机位、景别、推/拉/摇/跟/切镜/定格 |
| **格式约束** | 时长、画幅、节奏 | 开头声明时长；复杂度与秒数匹配，避免塞爆 |

### 分时段描述（时长 ≥ 8 秒时推荐）

```text
0–3秒：[开场画面、运镜、动作]
3–6秒：[中段发展]
6–10秒：[高潮或关键动作]
10–15秒：[收尾、定格]
```

5 秒短片：用 1–2 个镜头串联即可，不必强行分段。

### 运镜语言（择需使用，避免互相矛盾）

| 类别 | 示例 |
|------|------|
| 基础 | 推镜头、拉镜头、左摇/右摇、跟随镜头、环绕镜头、一镜到底 |
| 景别 | 面部特写、中景、全景、远景建立镜头 |
| 高级 | 低角度仰拍、俯拍、希区柯克变焦、快速摇镜 |

**禁止**在同一段同时要求「固定镜头」与「环绕镜头」等冲突指令。

### 风格修饰词（通常放在段末）

- 画面：`电影级质感，胶片颗粒，浅景深`、`2.35:1 宽银幕`、`超写实`、`冷暖对比`
- 氛围：`紧张悬疑`、`温暖治愈`、`纪录片风格`

### 官方常见避坑

1. **时长不匹配**：4–5 秒内不要塞入过多场景切换
2. **指令冲突**：运镜、节奏、动作要求需自洽
3. **叙事过载**：大段台词、内心独白模型无法直接拍出来
4. **多模态场景**（方舟平台）：@ 引用须写明用途；总参考文件 ≤ 12。**本项目筷子 API 为纯文生视频，不使用 @ 引用。**

## 本项目约束（筷子 OpenAPI / TextFlow）

1. 请求体仅含：`prompt`、`mode`、`resolution`、`ratio`、`duration`、`generation_type`。**不要**在 prompt 正文里写 model、API 参数或 JSON。
2. **默认 5 秒、竖屏 9:16**；内容多时可建议 10 秒，但仍须单条 prompt 内自洽。
3. **过滤器读整段场景意图**：避免真实品牌、版权 IP、真实名人、显性灾难暴力。
4. 输出**一条中文 prompt 正文**（150–400 汉字为宜），可直接填入 `prompt` 字段。
5. 应用运行时批量生成时，以本 skill 全文为 system，剧本正文为 user。

## 改写流程

```text
原文 → 风险扫描 → 去风险替换 → 剧本转电影化 → 按官方公式+四层补全 → 自检 → 输出
```

### Step 1：风险扫描

对照 [checklist.md](checklist.md)。常见触发源：

| 类别 | 示例 | 处理 |
|------|------|------|
| 真实品牌 | 茅台、iPhone、迪士尼 | 无标识白酒瓶、智能手机、卡通角色 |
| 版权 IP | 漫威、动漫名、游戏角色 | 外观描述，不出现作品/角色名 |
| 真实人物 | 明星、政客 | 年轻男职员、中年客户等虚构身份 |
| 显性灾难/暴力 | 蘑菇云、核爆、枪战、血腥 | 远处异常天光、云状光晕、对峙氛围 |
| 不良饮酒 | 拼酒、一饮而尽宣传 | 保留压力，弱化酗酒鼓动 |
| 剧本符号 | △、角色：、旁白、场次 | 删除，并入视觉描述 |
| 不可拍信息 | 内心独白、台词全文 | 表情、动作、镜头反应 |

### Step 2：剧本转电影化

删除剧本标记后，按**官方公式 + 四层**重写：

1. 开头：`[N]秒[画幅]电影感短片，[风格]，[光照/色调]。`
2. 主体 + 场景 + 动作（可见、可拍）
3. 运镜与转场（推近、切镜、定格等，二选一不冲突）
4. 可选：环境音/氛围音一句（不替代画面）
5. 合规收尾一句

### Step 3：输出模板（纯文生视频默认）

```text
5秒竖屏电影感短片，[风格]，[光照/色调]。[场景与陈设，无品牌]。
[镜头1：景别+机位+主体动作+可见细节]。
[镜头2或转场（可选）：随后/切向…]。
[可选]背景音：[环境声/氛围，一句]。
人物均为虚构面孔。无字幕、无品牌标识、无受版权保护角色。
不出现[本稿需规避的敏感具象]。
[可选]结尾定格在[画面]。
```

### Step 4：自检

输出前确认 [checklist.md](checklist.md) 全部通过。原文含高风险项时，附 3–5 条「已替换项」说明。

### Step 5：与项目集成

- 结果用于 `generateVideoB64` / `TextToVideoModal` 的 `prompt`
- 批量生成：每集剧本独立输出一条，不输出 JSON 包裹
- 用户要求「合并多场景」：用「镜头切向」「随后」快切串联，并注明画面过挤时可改 `duration=10` 或拆分两次生成

## 禁止事项

- 保留真实商标、产品全称、名人姓名
- 用 JSON/剧本/对白格式作为最终 prompt
- 用英文替换中文「绕过」审核
- 在 prompt 正文塞免责声明、API 说明、思考过程
- 假设未接入字段（negative_prompt、seed、@Image1 等）

## 降级策略

用户坚持高风险意象时，按序降级：

1. 具象 → 隐喻（蘑菇云 → 远处云状光晕）
2. 品牌 → 无标识同类物
3. 多场景 → 单场景最关键节拍
4. 仍失败 → 拆成两次生成或缩短 prompt
